# weread-to-feishu v1 改造计划

## 目标

把当前“仅支持覆盖写入飞书文档”的插件，改造成一个更合理的 v1：

- 免费主入口：**导出本地 Markdown**
- 保留现有能力：**同步到飞书文档（覆盖写入）**
- 在界面与文案上明确区分两种能力的定位与风险

---

## 改造原则

1. **先降门槛，再谈扩展**
   - 本地 Markdown 导出不依赖飞书配置，应成为默认低门槛入口。

2. **尽量少动现有稳定链路**
   - 飞书同步能力先保留，不做大重构，只做入口整理与风险提示。

3. **让产品表达更清晰**
   - 用户一眼能看懂：
     - 这个插件能干什么
     - 哪个能力适合先试
     - 飞书同步会覆盖原文档

4. **控制本轮改动范围**
   - 本轮不做：Notion、flomo、增量同步、账号体系、服务端。

## 实施步骤

### Step 1：补本地导出能力
新增本地导出模块，负责：
- 生成安全文件名
- 把 Markdown 内容下载到本地
- 支持多本书顺序导出

### Step 2：改造 popup 主界面
调整操作按钮与说明文案：
- 本地导出作为默认推荐动作
- 飞书同步保留，但强调“覆盖写入”

### Step 3：补充 manifest 权限
如采用浏览器下载 API，则补充 `downloads` 权限。

### Step 4：更新 README
让仓库首页能直接表达当前产品结构，而不是只像“飞书同步脚本”。

### Step 5：构建验证
运行：
- `npm install`
- `npm run build`

## 暂不处理的问题

以下问题确认存在，但不在本轮解决范围内：

- 飞书同步仍是“先删后写”的高风险模型
- 多本书仍未拆为“每本书一个飞书文档”
- 微信读书网页接口变动风险仍在
- Markdown -> Feishu Block 的转换能力仍较简化
- 没有增量同步、失败恢复、断点续传

---

## 完成本轮后的预期结果

完成后，插件会变成：

1. **普通用户可直接试用**
   - 登录微信读书网页版后，无需飞书配置即可导出 Markdown。

2. **飞书同步能力继续保留**
   - 面向有明确飞书沉淀需求的用户。

3. **产品结构更合理**
   - 免费入口更自然
   - 飞书能力更像高级用法
